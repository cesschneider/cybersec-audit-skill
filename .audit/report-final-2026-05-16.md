## Phase 2 — Manual Fix (2 findings)

### ✅ [HIGH-010] Broken Authentication — CWE-284 — server.js

**Was:** Static `X-Admin-Token` header check — single shared secret, no user identity.

**Fix implemented:**
- `POST /api/login` — validates credentials with bcrypt, issues signed JWT (8h expiry)
- `verifyToken` middleware — decodes JWT, populates `req.user = { id, username, role }`
- `requireAdmin` — chains `verifyToken` + `req.user.role === 'admin'` check
- Timing-safe login: always runs bcrypt.compare even for unknown users (prevents user enumeration)
- JWT_SECRET enforced at startup — process refuses to start if unset
- Bcrypt hashes stored in DB (cost factor 12)

**Test coverage:** 12 assertions covering valid login, wrong password, unknown user, missing fields, no token, invalid token, non-admin token, admin token.

---

### ✅ [MED-006] IDOR — CWE-639 — server.js:131

**Was:** `/api/profile/:userId` gated only by admin token — no ownership concept.

**Fix implemented:**
- `verifyToken` applied to `/api/profile/:userId`
- Ownership check: `req.user.id === targetId` OR `req.user.role === 'admin'`
- Regular users: can only access their own profile (403 otherwise)
- Admins: can access any profile
- Non-numeric userId → 400
- Non-existent userId → 404

**Test coverage:** 9 assertions covering unauthenticated access, own profile, cross-user access, admin access, edge cases.
